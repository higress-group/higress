## ai-proxy: 修复 Claude provider 的 thinking budget 未对 max_tokens 做 clamp 的问题

- 修复 `reasoning_effort` 映射为固定 `thinking.budget_tokens`（high=16384）时未与 `max_tokens` 比较，当 `max_tokens <= 16384` 时上游 Claude 返回 HTTP 400 `max_tokens must be greater than thinking.budget_tokens`。
- 新增统一的 `clampThinkingBudget` 逻辑：在标准 `thinking` 字段、`claude_thinking`、`reasoning_effort` 三个来源全部确定后，若 `budget_tokens >= max_tokens` 且 `max_tokens-1 >= 1024`，则将其钳制为 `max_tokens-1`，保证严格小于 `max_tokens`。
- 修复标准 OpenAI `thinking.budget_tokens`（复数）字段因 JSON tag 单数 `budget_token` 不匹配而被静默丢弃的问题（`provider/model.go` 的 `thinkingParam` 现同时兼容 `budget_token` 与 `budget_tokens`）。

fixes #4107
